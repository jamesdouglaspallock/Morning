ROLE:
You are a senior frontend engineer tasked with creating a **production-ready, Zillow/Apartments.com-style Property Details page**. The page must be **fully data-driven, professional, and mobile-first**.

ABSOLUTE RULES:
- Use **only fields confirmed in the backend audit**.
- Do not invent fields or hardcode values.
- All displayed data must come directly from the backend/property creation system.
- Maintain conditional rendering for empty or missing fields.
- Preserve mobile-first, high-conversion layout.

CONFIRMED BACKEND FIELDS:
- title, description
- address, city, state, zip_code
- price, bedrooms, bathrooms, square_feet
- year_built, property_type, furnished, pets_allowed, lease_term
- amenities (array), utilities_included (array)
- images (array, max 25)
- status, listing_status
- application_fee (must display dynamically)
- owner: full_name, email, phone
- Optional trust signals: price_history, view_count, save_count

---

PAGE STRUCTURE:

1️⃣ **IMAGE GALLERY (TOP)**
- Dynamic gallery from images[]: large main image + horizontal thumbnail strip
- Fullscreen modal on click with navigation
- Show image count overlay
- If images[] is empty, display “No photos available”

2️⃣ **PROPERTY SUMMARY (ABOVE THE FOLD)**
- Display: title, full address (address, city, state, zip_code)
- Display: price prominently ($/month)
- Display: bedrooms, bathrooms, square_feet
- Show status / listing_status clearly
- Display **Application Fee** prominently near price or Apply button
- Conditional rendering: hide fee if not set
- **Do NOT display views/save counts here**

3️⃣ **PRIMARY ACTIONS**
- Apply Now → opens application form (uses dynamic application_fee)
- Save Property → toggle saves to user favorites
- Share Listing → opens share modal
- Contact Owner → opens contact form using owner.email/phone
- Buttons must be fully functional

4️⃣ **OVERVIEW / DESCRIPTION**
- Render description from backend
- Preserve formatting
- Display “No description provided” if empty

5️⃣ **FACTS & FEATURES**
- property_type, year_built, lease_term
- furnished (Yes/No), pets_allowed (Yes/No)
- utilities_included[], amenities[] as checklist
- Only display fields that have values

6️⃣ **PRICE & TRUST**
- application_fee displayed here
- Optional trust signals (view_count/save_count) can be rendered elsewhere, not in fee section
- Optional: price_history graph

7️⃣ **LOCATION**
- Full address
- Map if latitude/longitude exist
- Do NOT use default/fallback coordinates

8️⃣ **OWNER / AGENT CONTACT**
- Display owner full_name, email, phone
- Wire contact form to backend inquiry logic

---

DESIGN REQUIREMENTS:
- Mobile-first, responsive, professional layout
- Clean typography, neutral palette
- Highlight key actions: Apply, Save, Share
- Gallery and buttons interactive on desktop & mobile
- Conditional rendering: hide empty data

VALIDATION CHECK:
- Every field rendered must exist in backend
- Application Fee is dynamic and prominent
- Views/save counts are separated from fee
- No placeholders or hardcoded values
- Buttons fully functional
